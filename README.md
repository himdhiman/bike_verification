# Bike Verification System

A Streamlit web application that verifies whether an uploaded bike image matches a registered bike model stored in a local database.

---

## Folder Structure

```
bike_verification/
├── dataset/                  ← Place your reference bike images here
│   ├── bike1.jpg
│   ├── bike2.jpg
│   └── bike3.jpg
├── bike_embeddings.npy       ← Auto-generated by generate_embeddings.py
├── generate_embeddings.py    ← Offline embedding builder
├── app.py                    ← Streamlit web application
├── requirements.txt
└── README.md
```

---

## How It Works

### Offline step (run once)
1. All images in `dataset/` are loaded.
2. Each image is passed through a pretrained **ResNet50** (final FC layer removed) to produce a **2048-dim feature vector**.
3. All vectors are saved to `bike_embeddings.npy`.

### Runtime step (Streamlit app)
1. User uploads an image.
2. The same ResNet50 pipeline generates an embedding for the uploaded image.
3. **Cosine similarity** is computed between the query embedding and every stored embedding.
4. The **maximum** score is taken.
5. `score > 0.75` → **VALID BIKE**, otherwise → **INVALID BIKE**.

## Setup & Run

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

> For GPU support, follow the [official PyTorch installation guide](https://pytorch.org/get-started/locally/) and install the CUDA-enabled build of `torch` and `torchvision`.

### 2. Add your reference images

Copy all images of the registered bike model into the `dataset/` folder:

```
dataset/
├── bike1.jpg
├── bike2.jpg
└── bike3.jpg
```

Supported formats: `.jpg`, `.jpeg`, `.png`, `.bmp`, `.webp`

### 3. Generate embeddings (run once, or whenever the dataset changes)

```bash
python generate_embeddings.py
```

Expected output:
```
Found 3 image(s). Loading ResNet50…
  Processing: dataset/bike1.jpg
  Processing: dataset/bike2.jpg
  Processing: dataset/bike3.jpg

Done. Saved 3 embedding(s) → 'bike_embeddings.npy'
Embedding matrix shape: (3, 2048)
```

### 4. Start the web app

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

---

## Configuration

Both scripts share a common threshold setting in `app.py`:

| Parameter             | Default | Description                                      |
|-----------------------|---------|--------------------------------------------------|
| `SIMILARITY_THRESHOLD`| `0.75`  | Minimum cosine similarity for a VALID BIKE result |

Increase the threshold to be stricter; decrease it to be more lenient.

## Tech Stack

| Component             | Library                          |
|-----------------------|----------------------------------|
| Web UI                | Streamlit                        |
| Feature extraction    | PyTorch + ResNet50 (torchvision) |
| Image preprocessing   | Pillow + torchvision transforms  |
| Similarity metric     | Cosine similarity (NumPy)        |
| Embedding storage     | NumPy `.npy` file                |
